# 프론트엔드 디자인 언어

목표: **"AI가 만든 티가 안 나는", 조용하고 밀도 있는 정리 도구.** 매일 열어도 질리지 않게 절제하되 밋밋하지 않도록 **약한 네오브루탈리즘**(굵은 먹 보더 + 딱딱한 오프셋 그림자)으로 촉감만 준다. 한국어 타이포그래피 중심.

핵심 구조: **중첩 폴더로 링크를 정리하고, 폴더 단위로 공유한다** (크롬 북마크 + 큐레이션). 태그는 보조(검색·필터).

## 0. 결정 이력

- 2026-07-22: 컬러 A안(먹+주홍) 확정 → **2026-07-23 이후 B안(먹+심록)으로 변경.** 주홍이 너무 강해 폐기.
- 레이아웃: 사이드바 대시보드 → **한국 웹식 가운데 고정폭 + 좌우 광고 레일**로 변경.
- 정리: 태그 평면 → **폴더 중심(중첩) + 태그 보조.**
- 촉감: "그림자 금지" → **약한 브루탈리즘**(하드 오프셋 그림자 허용, 소프트 blur 그림자는 여전히 금지).
- 카드: 16:9 썸네일 → **파비콘 컴팩트 카드**(썸네일은 선택 노출).

## 1. 하지 않을 것 (AI 티 목록)

- 보라→파랑 그라데이션, glassmorphism, 네온 글로우, **흐릿한 소프트 drop-shadow**(브루탈 하드 그림자와 구분)
- 이모지 남발, "✨ AI가 요약했어요!" 류 문구 — AI는 결과로만 존재하고 UI에서 떠들지 않는다
- 히어로 섹션 + 3D 일러스트 랜딩 — 로그인하면 바로 서랍이 열리는 도구다
- shadcn 기본 스킨 그대로 쓰기 (로직만 차용, 색·radius·간격은 자체 토큰)
- **원색 난립** — 사이트 파비콘 브랜드색을 그대로 카드에 쓰지 않는다(먹색 단색 타일로 통일). 랜덤 그라데이션 썸네일 금지.
- 큰 radius(12px+). 카드마다 16:9 썸네일 남발 — 텍스트 링크 정리 도구라 밀도를 죽인다.

## 2. 디자인 토큰

**타이포**: Pretendard Variable(본문/UI 전부). URL·도메인·시간·개수 등 메타데이터만 모노스페이스(JetBrains Mono 또는 시스템 mono, 10~12px). 이 대비가 "도구" 느낌을 만드는 핵심 디테일.
(목업 CSP상 Pretendard 임베드 불가라 시스템 폴백으로 보이지만, 실제 앱은 Pretendard Variable self-host.)

**컬러 — B안 "먹과 심록" (✔ 2026-07-23 확정)**:

| 토큰 | 값 | 용도 |
| --- | --- | --- |
| bg | `#FAF9F7` | 본문 지면 (웜 화이트, 종이) |
| bg-outer | `#F1EEE8` | 바깥 바닥 (광고 레일이 놓이는 면, 지면보다 살짝 어둡게) |
| surface | `#FFFFFF` | 카드 |
| ink | `#1C1B1A` | 본문 텍스트 |
| ink-muted | `#6E6A64` | 보조 텍스트, 메타 |
| line | `#E7E4DF` | 얇은 보더 (내부 구분, 1px) |
| brut-line | `#1C1B1A` | 굵은 보더 (카드·헤더 외곽, 2px) |
| shadow | `#1C1B1A` | 하드 오프셋 그림자 색 |
| accent | `#1F6A4E` | 심록 — 폴더 아이콘·활성 메뉴·저장 모드·포커스·링크 hover |
| accent-soft | `#E6F1EB` | 액센트 배경 (활성 칩 등) |

다크 모드: 토큰이 시맨틱이라 값만 스왑(accent `#4FA588`, bg `#1D1B1A`, brut-line `#5C574F`, shadow `#000` 등). Phase 5.

원칙:
- **평소 화면은 먹+종이 거의 단색. 심록은 상호작용/고정 자리에만 (전체 화면의 5% 미만).** 상태색은 실패(fail)만, 진행은 심록 pulse.
- radius **4px** 통일(칩만 full).
- 보더: 카드·헤더 외곽 **2px(brut-line)**, 내부 구분 1px(line).
- 그림자: **하드 오프셋만**(예: `2px 2px 0 shadow`, 흐림 0). 강조 요소(폴더 카드)에 기본, hover 시 살짝 떠오름. 소프트 blur 그림자 금지.

## 3. 레이아웃 (한국 웹식 가운데 고정폭)

```
┌───────────────────────────────────────────────────────────┐
│  즐겨모아   [내 서랍] [최근] [태그]              Prairie ● │  ← 전체폭 헤더(GNB)
├──────┬─────────────────────────────────────────────┬──────┤
│      │  ⌘K [ 검색하거나 URL 붙여넣기 ]   [검색/저장]│      │
│ 광고 │  🏠 내 서랍 / AI                             │ 광고 │  ← breadcrumb
│ 160  │  AI                     링크 82·폴더 6  [편집]│ 160  │
│ ×600 │  설명: RNN → LSTM → Transformer 순서로…      │ ×600 │  ← 폴더 헤더+설명
│      │  ─폴더─  [📁Transformer][📁Diffusion]…       │      │
│      │  ─링크─  [카드][카드][분석중]…               │      │
└──────┴─────────────────────────────────────────────┴──────┘
        └─ 본문 900px 고정, 가운데 정렬 ─┘
```

- 본문 컬럼 = **옴니바 → 폴더 헤더(breadcrumb+제목+설명) → 폴더 그리드 → 링크 그리드**.
- 좌우 **광고 레일 160×600, 선택**(안 붙여도 됨). 화면이 좁아지면(≤1180px) 숨고 본문만 가운데.
- 사이드바 없음(폴더 트리는 breadcrumb 이동으로 대체).

## 4. 폴더 & 시그니처 인터랙션

- **옴니바 하나로 검색·저장 겸용**(시그니처): 입력이 URL이면 "저장" 모드(Enter → **현재 폴더에** 카드 생성), 아니면 즉시 검색 필터. 오른쪽에 모드 표시(`저장 ↵` / `검색`). ⌘K 포커스. 별도 추가 버튼·모달 없음.
- **폴더 중심 이동**: 폴더 클릭 → 그 폴더 페이지로 **이동**(breadcrumb 늘어남). 인라인 펼침(아코디언) **안 함** — 한 번에 한 폴더만 로드(백엔드 부하 최소).
- **폴더 카드 (ver1, 강조)**: 굵은 보더 + 하드 그림자, hover 시 먹색 그림자로 살짝 떠오름(lift). 아이콘·이름·개수 + **파비콘 스택 미리보기**(안 열어도 뭐가 들었는지). 카드엔 설명 안 넣음(컴팩트 유지).
- **폴더 설명**: 폴더마다 자기 설명 1개. **소유주만 편집**(마우스 올리면 편집 버튼, 점선 테두리로 편집 가능 표시). 그 폴더에 들어가면 헤더에 크게 표시. 공유 시 받는 사람에게 **읽기 전용 안내문**으로 보인다(구글 드라이브 방식). 루트("내 서랍")엔 기본 설명이 깔림.
- **링크 카드 (ver2, 목록)**: 썸네일 없이 **파비콘(먹 타일) + 제목 2줄 + 요약 1줄 + 메타(도메인 mono·시간)**. 기본 그림자 없음, hover 시 **보더가 심록으로 + 배경 살짝 심록빛**(그림자 안 생김). 폴더 카드(그림자 lift)와 뚜렷이 구분.
- **분석 중 카드**: 스피너 대신 심록 pulse 점 + 스켈레톤 2줄. **클릭 가능**(진행 상세). 완료 시 fade-in 교체(150ms, 과한 스프링 금지).
- **실패 카드**: 좌측 5px fail 보더 + failReason 한 줄 + [재시도] 텍스트 버튼.
- 포커스: 심록 2px 링(키보드 내비 완전 지원). hover 시 lift/scale 남발 금지(폴더만 살짝).

## 5. 화면 목록 (MVP)

1. `/login` — 로고 + 구글 로그인 버튼 하나. 그게 전부인 화면.
2. `/` (루트 폴더 = "내 서랍") — 위 레이아웃. 폴더 헤더(기본 설명) + 폴더/링크 그리드. 빈 상태: "첫 링크를 붙여넣어 보세요".
3. 폴더 페이지 — 특정 폴더 안. breadcrumb + 그 폴더 설명(소유주 편집) + 하위 폴더 + 링크.
4. 카드 클릭 → 우측 슬라이드 패널(모달 아님): 요약/메모/태그/**폴더 이동** 인라인 편집, 원문 열기, 삭제.
5. 공유 공개 페이지 (읽기 전용) — 폴더 트리 + 링크 + 설명 안내문 + "내 서랍에 담기". **UI는 후순위**(데이터·구조는 미리 반영).
6. `/settings` — 태그 관리(rename/색/병합), 계정.

다크 모드는 Phase 5 (토큰이 시맨틱이라 값만 교체).

## 6. 문구 톤

- 한국어, 해요체 최소화 — 레이블은 명사형("저장", "재시도", "고정됨", "새 폴더").
- 에러는 원인 + 다음 행동: "본문을 가져오지 못했어요 · 재시도".
- AI 언급은 설정 화면 외에는 하지 않는다.
